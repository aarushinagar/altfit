## What Was Built

### 1. Good Morning Email (8:00 AM IST Daily)
**Route**: `POST /api/emails/good-morning`  
**File**: [app/api/emails/good-morning/route.ts](app/api/emails/good-morning/route.ts)

**Features**:
- Personalized morning greeting generated by Claude
- 6-piece wardrobe grid showcase (actual user photos)
- Grid layout: 77px × 100px cards with category + name labels
- CTA button to view full outfit on today page
- Deduplication: one per user per day
- Daily cron: `"30 2 * * *"` (2:30 AM UTC = 8:00 AM IST)

**Email Data**:
```typescript
interface GoodMorningEmailData {
  firstName: string;
  dayOfWeek: string;
  headline: string;        // Claude-generated
  bodyText: string;        // Claude-generated
  outfitPieces: Array<{    // Up to 6 wardrobe items
    name: string;
    category: string;
    imageUrl?: string;
  }>;
}
```

**User Query**:
- Filters: `emailOptOut: false`, `wardrobeItemCount >= 2`
- Fetches: 6 newest active wardrobe items ordered by creation date
- Selects: id, name, category, imageUrl

**Claude Prompt**: 
- Generates subject line (e.g., "Your Wednesday outfit awaits")
- Creates personalized morning greeting
- References user's style preference + wardrobe size
- Fallback: "Your outfit is ready." (if Claude fails)

### 2. Share Your Outfits Email (Wed 9:30 AM IST Weekly)
**Route**: `POST /api/emails/share-outfits`  
**File**: [app/api/emails/share-outfits/route.ts](app/api/emails/share-outfits/route.ts)

**Features**:
- 3-row share grid with emoji icons + CTAs
- Row 1 (📸): Share to Instagram
- Row 2 (💚): Share with Friends (WhatsApp)
- Row 3 (✨): Browse All Saved Outfits
- All CTAs link to `/saved-outfits` page
- Deduplication: one per user per week (prevents fatigue)
- Weekly cron: `"0 4 * * 3"` (4:00 AM UTC Wed = 9:30 AM IST Wed)

**Email Data**:
```typescript
interface ShareOutfitsEmailData {
  firstName: string;
  headline: string;       // Claude-generated
  bodyText: string;       // Claude-generated  
  savedCount?: number;    // User's saved outfit count
}
```

**User Query**:
- Filters: `emailOptOut: false`, `wardrobeItemCount >= 2`, at least 1 saved outfit
- Includes: saved outfit count via `_count.savedOutfits`
- Sends only once per 7 days per user

**Claude Prompt**:
- Generates community-focused subject (e.g., "Share your style")
- Creates sharing encouragement (self-expression focus, not growth)
- References saved outfit count
- Fallback: "Your saved looks tell a story" (generic encouraging version)

## Implementation Details

### Route Handler Pattern (Both Routes)

```typescript
1. ✅ Authorize request via timing-safe CRON_SECRET comparison
2. ✅ Fetch eligible users from Prisma:
   - Where: emailOptOut: false, wardrobeItemCount >= 2
   - Each route adds specific filters (saved outfits for weekly, etc)
3. ✅ Check deduplication in emailLog table
   - Good Morning: Skip if sent today to user
   - Share Outfits: Skip if sent in last 7 days
4. ✅ Build user context (firstName, styleProfiles, wardrobe count, etc)
5. ✅ Call personalizer: Claude generates personalized copy + fallback
6. ✅ Render template with data + inject unsubscribe token
7. ✅ Send via Resend API (RESEND_API_KEY env var)
8. ✅ Log to emailLog table:
   - emailType: "good_morning" or "share_outfits"
   - localDate: Date in user's timezone
   - resendId: UUID from Resend response
9. ✅ Return JSON response: { sent, failed, total, results }
```

### Error Handling

Both routes implement:
- Bearer token validation with `isCronAuthorized()` helper
- Try/catch blocks around Prisma queries
- Try/catch blocks around Claude API calls
- Resend send errors caught and logged
- Graceful fallback copy if Claude fails
- Detailed error responses for debugging
